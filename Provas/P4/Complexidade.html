Q1)

Explique as seguintes afirmações e questionamentos:

(a) A função f(n)f(n)f(n)﻿ tem complexidade O(n2)O(n^2)O(n2)﻿.

  • Isso significa que o tempo de execução ou o número de operações da função f(n)f(n)f(n)﻿ cresce, no pior caso, no máximo como n2n^2n2﻿, quando o tamanho da entrada nnn﻿ aumenta.
  • A notação OOO﻿ (letra "O" maiúscula), chamada de notação "big-O", expressa um limite superior assintótico.
  • Exemplo: se f(n)=3n2+2n+5f(n)=3n^2+2n+5f(n)=3n2+2n+5﻿, dizemos que f(n)f(n)f(n)﻿ ∈ O(n2)O(n^2)O(n2)﻿, pois, à medida que n→∞n \to \inftyn→∞﻿, o termo dominante é n2n^2n2﻿, e os outros se tornam irrelevantes.

(b) O tempo necessário para execução do algoritmo tem complexidade Θ(n log n)

  • A notação Θ\ThetaΘ﻿ (Theta) indica que o tempo de execução do algoritmo cresce exatamente na ordem de nlog⁡nn \log nnlogn﻿. Ou seja, nem mais rápido nem mais lento que isso assintoticamente.
  • É uma limitação tanto superior quanto inferior do comportamento da função.
  • Exemplo clássico: algoritmos como Merge Sort e Heap Sort têm complexidade Θ(nlog⁡n)\Theta(n \log n)Θ(nlogn)﻿, pois esse é o número mínimo e máximo de comparações em todos os casos.

(c) Qual o algoritmo mais veloz, o que tem complexidade O(n) ou um outro com Θ(n2)Θ(n^2)Θ(n2)﻿ ?

  • Um algoritmo com complexidade O(n)O(n)O(n)﻿ (linear) é, assintoticamente, mais rápido do que um com Θ(n2)\Theta(n^2)Θ(n2)﻿ (quadrática), especialmente para valores grandes de nnn﻿.
  • Por quê?
    • O(n)O(n)O(n)﻿: o tempo cresce proporcionalmente ao tamanho da entrada.
    • Θ(n2)\Theta(n^2)Θ(n2)﻿: o tempo cresce proporcional ao quadrado do tamanho da entrada.
  • Por exemplo:
    • Se n=1000n=1000n=1000﻿:
      • O(n)≈1000O(n) \approx 1000O(n)≈1000﻿ operações.
      • Θ(n2)≈1.000.000\Theta(n^2) \approx 1.000.000Θ(n2)≈1.000.000﻿ operações.
Portanto, o algoritmo com complexidade O(n)O(n)O(n)﻿ é geralmente mais eficiente do que o de Θ(n2)\Theta(n^2)Θ(n2)﻿, a menos que o primeiro tenha um custo constante muito alto (o que é raro).

Q2)

Quais as complexidades de tempo dos algoritmos abaixo (big O):

(a) A1A_1A1​﻿: Ordenação de uma lista sequencial não ordenada;

  • Resposta: Depende do algoritmo de ordenação usado.
    • Bubble Sort, Insertion Sort, Selection Sort:
      • O(n2)O(n^2)O(n2)﻿
    • Merge Sort, Heap Sort, Quick Sort (melhor caso):
      • O(nlog⁡n)O(n \log n)O(nlogn)﻿
  • Como a questão não especifica o algoritmo, assume-se o pior caso possível para ordenação genérica:
    • O(nlog⁡n)O(n \log n)O(nlogn)﻿ (usando um algoritmo eficiente como Merge Sort)

(b) A2A_2A2​﻿: Busca de um elemento em uma pilha formado por lista encadeada.

  • Pilha: estrutura LIFO (Last-In, First-Out)
  • Para buscar um elemento arbitrário, é necessário percorrer a pilha elemento por elemento.
    • O(n)O(n)O(n)﻿ (no pior caso, o elemento está no final ou não está)

(c) A3A_3A3​﻿: Busca de elementos em uma lista linear encadeada ordenada;

  • Mesmo sendo ordenada, não é possível fazer busca binária em lista encadeada linear, pois não há acesso direto a posições intermediárias.
  • Portanto, a busca ainda é feita linearmente.
    • O(n)O(n)O(n)﻿ (mesmo sendo ordenada)

(d) A4A_4A4​﻿: Inserção de elemento numa fila formado por lista encadeada;

  • Inserção em uma fila é feita no final (enfileirar).
  • Se a fila for implementada com ponteiro para o fim (cauda):
    • O(1)O(1)O(1)﻿ (inserção direta)
  • Se não tiver ponteiro para o fim:
    • O(n)O(n)O(n)﻿ (teria que percorrer até o final)
  • Assumindo uma implementação eficiente:
    • O(1)O(1)O(1)﻿

(e) Remoção de elemento em uma lista sequencial;

  • Lista sequencial = vetor
  • Para remover um elemento de qualquer posição:
    • É preciso deslocar os elementos à direita uma posição à esquerda.
  • Portanto:
    • O(n)O(n)O(n)﻿ (no pior caso, remove-se o primeiro elemento)

Q3)

Quais as complexidades de memória dos algoritmos abaixo (big O):

(a) Inserção de elementos em uma pilha;

  • Cada nova inserção (push) aloca memória para um novo elemento.
  • Portanto, para n elementos, o uso de memória cresce linearmente.
  • Complexidade de memória: O(n)O(n)O(n)﻿

(b) Inserção de elementos em uma fila;

  • Também cresce linearmente com o número de elementos inseridos.
  • Em implementações com lista encadeada ou vetor com redimensionamento automático, o espaço necessário é proporcional ao número de elementos.
  • Complexidade de memória: O(n)O(n)O(n)﻿

(c) Remoção de elementos em uma pilha;

  • Ao remover (pop), o elemento é simplesmente descartado.
  • Isso não aumenta o uso de memória, apenas reduz.
  • A operação em si não consome memória adicional, exceto espaço constante (temporário).
  • Complexidade de memória: O(1)O(1)O(1)﻿

(d) Remoção de elementos em uma fila;

  • Assim como na pilha, remover (dequeue) não consome memória adicional.
  • Libera espaço (dependendo da implementação), mas não cresce com nnn.
  • Complexidade de memória: O(1)O(1)O(1)﻿

Q5)

Para decidir qual algoritmo utilizar, devemos comparar as complexidades assintóticas:

Algoritmo A: O(n5)O(n^5)O(n5)﻿

  • Complexidade polinomial.
  • Cresce rapidamente, mas ainda é tratável para valores razoáveis de nnn.

Algoritmo B: O(1.5n)O(1.5^n)O(1.5n)﻿

  • Complexidade exponencial (base 1.5).
  • Cresce muito mais rapidamente do que qualquer função polinomial, mesmo com base pequena.
  • Para valores pequenos de nnn, o algoritmo B pode parecer mais rápido.
  • Mas, à medida que nnn cresce, 1.5n1.5^n1.5n﻿ ultrapassa n5n^5n5﻿ rapidamente e torna-se inviável.

Conclusão:

  • Para valores pequenos de entrada, algoritmo B pode ser aceitável.
  • Mas em geral, o algoritmo A (com O(n5)O(n^5)O(n5)﻿) é melhor, porque:
    • Complexidade polinomial é muito mais escalável do que exponencial.
    • Algoritmos exponenciais se tornam impraticáveis muito rapidamente.

Portanto, eu usaria o algoritmo A.

Q7)

Calcule a complexidade do algoritmo abaixo:

int f(int n) {
    int s = 0;
    for (int i = 0; i < n; i++)              // (linha 3)
        for (int k = n; k < i; k++)          // (linha 4)
            s = s + i;                       // (linha 5)
}

Análise dos loops:

  • O laço externo (linha 3) vai de i = 0 até i < n, ou seja, roda n vezes.
  • O laço interno (linha 4) vai de k = n até k < i.

Mas perceba um detalhe importante:

  • Quando i < n, sempre temos que k = n > i, então a condição k < i nunca é satisfeita.
  • Ou seja, o laço interno nunca é executado porque o valor inicial de k (que é n) já não satisfaz a condição k < i.

Exemplo:

Se n = 5, então:

  • Quando i = 0, k = 5, mas 5 < 0 é falso
  • Quando i = 1, k = 5, 5 < 1 é falso
  • ...
  • Isso vale para qualquer i < n

Conclusão:

  • O laço externo executa nnn vezes.
  • O laço interno não executa nenhuma vez.
  • A única operação efetiva é a atribuição int s = 0; (linha 2), que é constante.

Complexidade final:

  • O corpo do laço nunca é executado.
  • Portanto, a função sempre faz apenas operações constantes.

Resposta final:

A complexidade do algoritmo é O(1)\boxed{O(1)}O(1)​﻿

Q12)

Computador 1 → Algoritmo A (O(n2)O(n^2)O(n2)﻿);

Computador 2 → Algoritmo B (O(nlog⁡(n))O(n \log (n)) O(nlog(n))﻿);

Sobre esses algoritmos e suas execuções, pode-se afirmar que:

(a) ( ) O computador 1 é mais lento que o computador 2;

FALSO.

A lentidão ou rapidez do computador depende do hardware, sistema operacional, etc., e não é possível inferir isso apenas com base no algoritmo executado.

(b) ( ) Para uma mesma quantidade de dados, o computador 2 geralmente conclui a ordenação antes do 1;

VERDADEIRO.

O algoritmo B, com complexidade O(nlog⁡n)O(n \log n)O(nlogn)﻿, tende a ser mais eficiente do que o algoritmo A, de complexidade O(n2)O(n^2)O(n2)﻿, para valores grandes de n. Isso porque o crescimento de n2n^2n2﻿ é bem mais rápido do que o de nlog⁡nn \log nnlogn﻿, então, em geral, o algoritmo B termina primeiro.

(c) ( ) Se o computador 1 tiver o dobro de memória do computador 2, então o algoritmo A será mais eficiente;

FALSO. A eficiência assintótica está relacionada ao tempo de execução em função do tamanho do problema, não à memória. Ter mais memória pode ajudar, mas não muda a complexidade O(n2)O(n^2)O(n2)﻿.

(d) ( ) A complexidade é dependente da configuração do computador;

FALSO. A complexidade (Big O) mede o comportamento teórico do algoritmo com o crescimento da entrada, independente do computador. O que pode mudar é o tempo exato de execução, não a classe de complexidade.

(e) ( ) O algoritmo 1 é mais lento que o algoritmo 2, dependendo da quantidade de dados e independente do computador.

VERDADEIRO. Para entradas pequenas, o algoritmo O(n2)O(n^2)O(n2)﻿ pode até ser mais rápido se for mais simples. Mas com o crescimento de n, o algoritmo com O(nlog⁡n)O(n \log n)O(nlogn)﻿ se torna significativamente mais rápido. Isso ocorre independentemente do computador, pois é uma questão da forma como o algoritmo cresce com n.
